# Context Switching Protocol

**Created:** 2025-09-04 02:02
**Type:** #agent-protocol #context-management
**Status:** #status/active

## Protocol Overview
Standardized process for maintaining context continuity when switching between projects, domains, or sessions in the Claude Code + Obsidian ecosystem.

## Pre-Switch Checklist
### Current Context Preservation
- [ ] **Session Summary:** Document current task state and progress
- [ ] **Decision Log:** Record key decisions made and rationale
- [ ] **Pattern Observations:** Note any new patterns or insights discovered
- [ ] **Blockers & Issues:** Document unresolved problems and context
- [ ] **Next Steps:** Clear handoff notes for continuation

### Memory System Updates
- [ ] **Daily Memory:** Update with session outcomes
- [ ] **Project Memory:** Save project-specific context and status
- [ ] **Pattern Library:** Add any new workflows or solutions discovered
- [ ] **Knowledge Graph:** Update connections and insights

## Context Loading Protocol
### Target Context Preparation
1. **Identify Target Domain/Project**
   - Load relevant project memory file
   - Review previous session logs
   - Check current project status and priorities

2. **Refresh Mental Model**
   - Review project objectives and constraints
   - Load domain-specific knowledge and patterns
   - Identify key stakeholders and resources

3. **Set Session Parameters**
   - Define session objectives and scope
   - Establish success criteria
   - Identify potential challenges and resources

### Agent Specialization Setup
- **Determine Primary Agent Role:** Based on task type and domain
- **Load Specialized Context:** Domain-specific patterns and knowledge
- **Configure Tool Preferences:** Optimal tool selection for domain
- **Set Communication Style:** Appropriate for context and objectives

## Switch Execution
### Context Transition
```markdown
## Context Switch: [Timestamp]
**From:** [Previous Context]
**To:** [New Context]
**Switch Reason:** [Why switching]
**Handoff Notes:** [Critical information to preserve]
**Estimated Duration:** [Expected time in new context]
```

### Immediate Actions
1. **Context Validation:** Confirm understanding of new context
2. **Objective Alignment:** Verify goals and expectations
3. **Resource Check:** Ensure necessary tools and information available
4. **Quick Win Identification:** Find immediate value-add opportunities

## Context Types & Specific Protocols

### Project Context Switches
**Use When:** Moving between different projects or major initiatives

**Specific Steps:**
- Load project charter and current status
- Review recent project-specific sessions
- Check project-specific agent memory
- Align with project timeline and milestones
- Identify project-specific patterns and constraints

### Domain Context Switches  
**Use When:** Moving between different knowledge domains or skill areas

**Specific Steps:**
- Load domain-specific knowledge base
- Review domain patterns and best practices
- Adjust communication style for domain
- Check domain-specific tools and resources
- Identify cross-domain connection opportunities

### Temporal Context Switches
**Use When:** Returning to work after break or starting new session

**Specific Steps:**
- Review previous session outcomes
- Check daily/weekly objectives
- Update on external changes or new information
- Refresh on current priorities
- Validate assumptions and contexts

## Quality Assurance
### Context Integrity Checks
- [ ] **Completeness:** All necessary context loaded?
- [ ] **Currency:** Information up-to-date and relevant?
- [ ] **Consistency:** No conflicting information or objectives?
- [ ] **Clarity:** Clear understanding of objectives and constraints?

### Performance Metrics
- **Switch Time:** Duration of context switching process
- **Context Fidelity:** How completely context was preserved/loaded
- **Productivity Impact:** Effect on task performance post-switch
- **Error Rate:** Mistakes or misunderstandings due to context issues

## Optimization Strategies
### Efficiency Improvements
1. **Template Utilization:** Standardized context switch templates
2. **Automated Logging:** Scripts to capture and restore context
3. **Quick Reference:** Essential context summaries for rapid loading
4. **Pattern Recognition:** Identify common switch patterns for optimization

### Context Compression
- **Essential vs Nice-to-Have:** Focus on critical context elements
- **Hierarchical Loading:** Load high-level first, drill down as needed
- **Just-in-Time Context:** Load detailed context when specifically needed
- **Context Caching:** Pre-load frequently accessed contexts

## Emergency Protocols
### Context Loss Recovery
If context is lost or corrupted:
1. **Immediate Assessment:** What context remains intact?
2. **Rapid Reconstruction:** Use available logs and memory files
3. **Stakeholder Communication:** Inform of context loss and recovery plan
4. **Preventive Measures:** Implement additional backup strategies

### Context Conflict Resolution
When contexts conflict or compete:
1. **Priority Assessment:** Determine which context takes precedence
2. **Conflict Documentation:** Record nature and resolution of conflict
3. **Integration Strategy:** Find ways to harmonize conflicting contexts
4. **Future Prevention:** Update protocols to prevent similar conflicts

## Advanced Context Management
### Multi-Context Awareness
Maintaining awareness of multiple contexts simultaneously:
- **Primary Context:** Main focus with full detail
- **Secondary Context:** Background awareness for integration opportunities  
- **Tertiary Context:** Minimal awareness for potential relevance

### Context Bridging
Creating connections between different contexts:
- **Pattern Transfer:** Apply patterns from one context to another
- **Resource Sharing:** Leverage resources across contexts
- **Insight Synthesis:** Generate new insights from context combinations
- **Cross-Pollination:** Encourage innovation through context mixing
